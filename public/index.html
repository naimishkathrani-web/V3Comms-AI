Core Endpoints

  • POST /api/knowledge/intake/upload create a file draft and AI recommendation
  • POST /api/knowledge/intake/url fetch a URL, classify it, and create a draft
  • POST /api/knowledge/intake/:id/review approve role/category/sub-category/company/project/commodity metadata
  • POST /api/knowledge/intake/:id/ingest push the reviewed record into pgvector
  • POST /chat auto-infers role/category when none is supplied

What happens on ingest

  • The record is stored in the intake ledger first.
  • AI recommends role, category, sub-category, company, project, commodity, tags, and a short summary.
  • The approved document is chunked and embedded with nomic-embed-text.
  • Chunks are written into pgvector with metadata filters.
  • The record becomes read-only after ingestion.
